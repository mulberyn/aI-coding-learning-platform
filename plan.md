# 项目计划

## 要求1

先完成学生（普通用户）的页面显示，并且留出教师、管理的修改空间，可拓展性高，要求项目目录结构清晰：

1. 优化顶部导航栏，顶部导航栏应该有：首页（用户首页）、题库、提交记录、比赛、论坛。最右侧分别是登陆/注册按钮（如果已经登陆则显示头像）和明暗切换按钮。

顶部导航栏在整体页面下拉的过程中会消失，上拉时会重新出现，消失和出现过程中有丝滑的动画效果。在对应页面中，顶部导航栏的对应选项下方有一个小椭圆横线，表示当前所在页面。切换到其他页面时，顶部导航栏的对应选项下方的小椭圆横线会有一个动画效果，平滑地移动到新的位置。

2. 优化题目列表

题目列表顶部左侧包括一个搜索框和一个标签筛选器。搜索框允许用户输入搜索题目，标签筛选器允许用户根据题目的标签进行筛选。顶部右侧有一个是否显示标签的开关，用户可以选择是否显示题目的标签信息。

题目列表以横向行的形式展示，每一行之间不要有间距，依次左靠齐显示题目编号，题目名称，右靠齐显示过题人数，难度等级（简单、普通、困难）、通过率。题目名称可以点击，点击后会进入题目详情页。

登陆后，每个题目行会在题目编号的左上角显示一个左上三角形（三角形中有一个小五角星），点击后用户会收藏该题目，再次点击会取消收藏。

## 要求2

重构顶部导航栏的组件代码。

将顶部导航栏的组件从 `components` 目录移动到 `app` 目录下，放在一个新的文件夹 `components` 中，路径为 `app/components/TopNavBar.tsx`。

删除不同页面中的所有解释话语，比如在 `app/login/page.tsx` 中删除以下内容：

```typescript
    <SiteShell
        title="首页"
        eyebrow="学生主入口、学习概览和平台能力总览"
        requireAuth={false}
        >
```

顶部导航栏最左侧为一个 OJ 的 logo（暂定为 AIOJ），点击后会跳转到首页。导航栏中包括：首页（用户首页）、题库、提交记录、比赛、论坛。最右侧分别是登陆/注册按钮和明暗切换按钮。样式参考 LOJ（https://loj.ac/）的顶部导航栏。

取消明显的卡片外框，要求导航栏的内容等高显示，选项之间没有明显的分割线，整体风格简洁现代。

登陆/注册按钮为两个方形按钮，按钮四个角做微小圆角处理，登陆按键为灰色，注册按钮为浅蓝色。在用户未登录时显示，点击后会跳转到登录/注册页面；如果用户已经登录，则显示用户头像，点击后会在下方出现一个菜单，包含“个人中心”、“设置”和“退出登录”选项。

明暗切换按钮为一个正方形按钮，按钮四个角做微小圆角处理。允许用户在浅色模式和深色模式之间切换，切换后会立即应用新的主题，正常情况是电视图案表示系统默认选项、太阳图案表示浅色模式、月亮图案表示深色模式。图标要求符合整体风格，简洁现代。

在对应页面中，顶部导航栏有阴影效果，并且对应选项下方有一个小椭圆横线，对应当前所在页面。切换到其他页面时，顶部导航栏的对应选项下方的小椭圆横线会有一个动画效果，平滑地移动到新的位置。

顶部导航栏是一个整体，导航栏最下方的横线下有细的阴影效果。

## 要求3

导航栏每个选项底的黑色横线（粗横线，不是卡片样式）在切换页面时有动画效果，平滑地移动到新的选项处。实现这个动画效果可以使用 CSS 的 transition 属性来控制小椭圆横线的位置变化，使其在切换页面时平滑地移动到新的位置，并且在对应页面的选项后用阴影效果突出显示当前所在页面。动画效果应该流畅自然，提升用户体验。

每个选择前添加一个小图标，图标要求符合整体风格，简洁现代。可以使用一些常见的图标库，如 Font Awesome、Material Icons 等，选择适合的图标来表示每个选项。例如：首页前添加一个房子图标，题库前添加一个书本图标，提交记录前添加一个文件夹图标，比赛前添加一个奖杯图标，论坛前添加一个对话框图标。

选项底部的横线动画从原位置丝滑移动到新位置，动画持续时间为 0.3 秒，使用 ease-in-out 的缓动函数，使动画效果更加自然流畅。阴影更加大一点，颜色可以稍微深一点，以突出显示当前所在页面。

给 AIOJ 前添加一个 logo 图标，字体换一个稍微粗一点的现代风格字体。
