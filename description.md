【用户管理模块】
本系统采用 NextAuth + Credentials 的鉴权方案，实现注册、登录、会话维持与权限识别。用户注册阶段通过 zod 对昵称、邮箱、密码进行校验，并使用 bcrypt 对密码进行哈希存储；登录阶段基于邮箱与密码进行身份验证，登录成功后写入 JWT 会话。系统在会话回调中扩展了 userId 与 role 字段，支持 STUDENT、TEACHER、ADMIN 三类角色的能力区分，为后续页面访问控制和数据权限判断提供基础。个人中心与设置页支持用户查看基础资料、学习画像与统计信息，并维护 AI 相关参数。平台还提供用户级 API Key 配置管理能力，支持多条大模型配置的新增、查询、启用和删除；启用某配置时会自动将其余配置置为非激活，保证系统始终只有一个生效配置。相关接口均进行会话身份比对与越权拦截，确保“仅本人可操作本人数据”，整体形成了从账号生命周期管理到个性化配置管理的完整闭环。

【题库系统模块】
题库模块围绕 Problem、Example、TestCase 三类核心数据构建，支持题目标题、题面描述、难度、题型、函数签名、输入输出格式、时间与内存限制等完整要素建模。前端提供题库列表与题目详情页面，后端通过 Prisma 查询实现按题型与难度组织展示，并返回示例与统计信息。系统支持函数式题与传统输入输出题两种模式，满足不同 OJ 场景。为提升数据真实性，题库统计不直接使用静态字段，而是基于 Submission 进行 groupBy 聚合，动态计算尝试人数、通过人数与通过率。模块同时维护用户题目状态映射（未尝试/已尝试/已解决），便于在列表中展示学习进度与做题轨迹。

【代码评测系统模块】
代码评测模块以 Submission 与 JudgeResult 为核心，形成“提交入库—异步评测—结果回写—明细查询”的完整链路。用户提交代码时，系统先进行登录校验与参数校验，再校验题目及测试用例有效性；提交成功后创建提交记录，并按测试点预生成评测结果占位记录。评测执行由 runSubmissionJudging 异步触发，状态从 QUEUED 进入 RUNNING，逐个测试点调用 Judge0 远程执行服务完成编译与运行。系统内置多语言映射（C/C++/Python/Go/Rust/Java），并实现重试与退避机制，应对 503/504/429 等临时故障；轮询阶段根据 Judge0 状态码归一化为业务状态（通过、答案错误、编译错误、运行错误、超时等），最终聚合得到整次提交状态、评分与提示信息。接口层还提供提交详情查询，支持返回源码、测试点执行日志、耗时和内存数据，并结合角色权限控制可见范围，保证教学管理与用户隐私的平衡。

【竞赛管理系统模块】
竞赛模块围绕 Contest、ContestProblem、ContestRanking、ContestRegistration 数据模型实现比赛全流程管理。系统支持官方赛、公开赛、回放赛等类型及 OI/ICPC/IOI 赛制字段，记录起止时间、时长、公告、参赛人数、题目编排和排行榜明细。前端提供比赛列表与详情页，支持查看进行中/历史比赛、赛题顺序与榜单信息。用户报名与退赛通过独立 API 完成，并在事务中同时维护报名表与 participantCount 计数，保证并发下数据一致性。业务规则上仅允许未开始比赛进行报名或取消，且对重复报名、未报名取消等场景进行明确拦截，体现了较完整的状态机约束与竞赛治理能力。

【AI智能辅导模块】
AI 智能辅导模块是平台个性化学习能力的核心，采用“可配置模型接入 + 多场景辅导服务 + 对话持久化”架构。系统支持 Deepseek、Qwen 两类模型服务，用户可在个人设置中维护多组 API Key 配置并指定激活项；当存在激活配置时优先使用，否则回退到用户默认模型参数。模块提供通用对话接口，可接收多轮消息并结合系统提示词生成教学化回答，支持 Markdown 与公式输出。针对评测场景，平台实现代码分析、改进建议、错误解析三类专项辅导，自动读取提交代码、题目信息与判题状态构造提示词，生成结果后写入 AiTutoring 表并累计辅导次数。为支持连续学习，系统还实现对话历史存储与删除能力，按用户隔离会话数据。进一步地，模块结合用户近期提交、论坛参与与竞赛记录，提取薄弱主题、通过情况与行为特征，自动生成学习路线建议 JSON，并在解析失败时回退到内置模板，保证服务稳定可用。该模块将“答疑、诊断、建议、规划”整合为统一智能学习助手，为毕设中的“AI 赋能教学”提供了可落地实现。

【论坛系统模块】
论坛模块提供发帖、回帖、检索与分版管理能力，数据层由 ForumPost 与 ForumComment 组成，支持站务、求职、学术、题目四类板块。发帖接口基于 zod 校验标题、内容与版块字段，并对题目版进行额外约束：必须绑定题号且题号需能映射到真实题目，避免无效关联。评论接口支持按帖子新增回复并返回结构化结果。前端论坛页提供关键词搜索、题号过滤、版块筛选与分页浏览，列表按“置顶优先 + 时间倒序”排序，并展示作者、时间、回复数和关联题号信息。整体设计实现了学习社区中“问题提出—讨论互动—经验沉淀”的基本闭环。

【学习路线系统模块】
学习路线模块支持用户创建、查看、更新与删除个性化学习计划，采用学习路线主表与学习点子表的二级结构组织数据。系统在运行时自动检测并创建 learning_routes 与 learning_route_points 表，保证在轻量数据库环境下快速落地。每条路线包含来源、输入提示、总结与多个学习点，学习点支持题目/比赛/论坛/自定义四类类型，并维护目标日期、状态与排序。接口层提供路线列表查询、详情查询、学习点状态更新等能力，并对标题长度、描述长度、点数量做安全裁剪。结合 AI 推荐接口，系统可依据用户目标与历史行为生成 4-8 个按时间推进的学习步骤，形成“自动生成 + 人工调整 + 进度跟踪”的学习规划机制。

【统计分析系统模块】
统计分析模块从题目维度与用户维度两条线构建学习画像。题目维度通过 /api/statistics/[slug] 聚合该题所有提交，计算总提交数、通过数、通过率，并按结果类型统计分布（通过、答案错误、运行错误、超时、编译错误）；同时从判题明细中提取平均运行时间、平均内存与代码长度等指标，为题目难点分析和性能对比提供依据。用户维度基于近周期提交行为构建周度指标序列，包括完成题目数、知识点覆盖、AI 辅导轮次和薄弱模块数量，并计算周环比变化；个人中心进一步通过热力图展示最近一年活跃度，直观呈现学习连续性。系统还能结合薄弱主题与行为数据生成 AI 周评语与推荐内容，实现“数据统计—问题发现—建议输出”的闭环分析流程，为教学评估与自适应学习提供支撑。
